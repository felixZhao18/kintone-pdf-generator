# 業務報告書アプリ（PDFプレビュー＆印刷プラグイン付き）

kintone上で「報告書」を作成・管理し、ボタン一つでA4レイアウトのPDFを生成してブラウザ上で閲覧・印刷できるカスタマイズ一式です。

## 概要 / アプリの特徴
- **プロセス管理連動**：一次承認者 ➔ 最終承認者の順で回る承認ワークフローを搭載。
- **ワンクリックPDF生成**：詳細画面に設置された「PDFプレビューを表示する」ボタンを押すと、印刷用のA4サンドボックスを展開し、ズレのないPDFを即座に生成します。
- **動的レイアウト**：作成者や日時などの基本情報をテーブル形式で綺麗に整列。「感想」や「備考」は、入力がある場合のみPDF上に自動で枠線付きで表示されます。

## 倉庫にあるファイル / 含まれるファイル
- `index.js`：kintone用のメイン制御ソースコード。
- `html2pdf.bundle.min.js`：PDF生成を高速化（ローカルキャッシュ化）するための軽量ライブラリ。
- `kintone_app_template.zip`：本アプリのテンプレート数据。インポートするだけで同じアプリが作れます。

## 必要となるkintoneのフィールド設定
このカスタマイズを動作させるため、アプリ内には以下の**フィールドコード**が正しく設定されている必要があります：

| フィールド名 | フィールドタイプ | フィールドコード (必須) |
| :--- | :--- | :--- |
| アプリタイトル | 文字列__1行 | `アプリタイトル` |
| 対象 | 文字列__1行 | `対象` |
| 実施場所 | 文字列__1行 | `実施場所` |
| 実施開始日 | 日付 | `実施開始日` |
| 実施終了日 | 日付 | `実施終了日` |
| 報告内容 | リッチエディター | `リッチエディター` |
| 感想 | 文字列__複数行 | `感想`|
| 備考 | 文字列__複数行 | `備考` |

## 導入・操作マニュアル (同僚の方へ) / 部署部署指南

他ノアプリや独自の環境にこの機能を導入する場合、以下の手順に従って設定を行ってください。

### 1. 外部ライブラリ（html2pdf）の準備
PDF生成エンジンとして `html2pdf.bundle.min.js` が必要です。
- **ダウンロード方法**: ブラウザで下記の公式CDNリンクを開き、画面上で右クリックして「名前を付けて保存」でローカルに保存してください。
  > `https://cdn.jsdelivr.net/npm/html2pdf.js@0.10.1/dist/html2pdf.bundle.min.js`
  *(※ または、本GitHubリポジトリから直接ダウンロードしても構いません)*

### 2. kintoneへのアップロード設定
1. 対象のkintoneアプリの設定 ＞ **「JavaScript / CSSでカスタマイズ」** を開きます。
2. **「PC用のJavaScriptファイル」** の項目に、ファイルをアップロードします。

#### 【最重要】アップロードの順序について
kintoneは上に配置されたファイルから順番に実行されるため、**順序が逆になるとPDF機能がエラーで動作しません。** 必ず以下の順序になっていることを確認してください：

- **1番目（上側）**: `html2pdf.bundle.min.js` （外部ライブラリ）
- **2番目（下側）**: `index.js` （メインロジックコード）

>  **チェック**: 順番が違っている場合は、右側にある「⬆️ ⬇️」ボタンを使って順序を入れ替えてください。

3. ファイルの追加が完了したら、左上の **「保存」** ボタンを押し、次の画面の右上にある青い **「アプリを更新」** ボタンをクリックして適用します。
